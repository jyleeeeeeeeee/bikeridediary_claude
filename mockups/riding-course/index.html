BRD 라이딩 코스 기능 목업

Flutter 구현 전 시각적 확인용 HTML 목업입니다.
조회 + 즐겨찾기 + 소유마커까지 포함 (코스 생성/편집은 2차 스코프, 이번 목업에서 제외).

화면 목록

🏠
화면 01
홈 확장 메뉴 (4버튼)
기존 3버튼에서 "라이딩 코스" 버튼 추가. 부채꼴 열림 상태.
❤️
화면 02
라이딩 코스 홈 — 내 코스 탭
내가 만든 코스(하트) + 즐겨찾기한 코스(별)의 혼합 리스트. FAB(+) 포함.
🔍
화면 03
라이딩 코스 홈 — 탐색 탭
검색창 + 전체 코스 리스트 + 즐겨찾기 별 토글. 클릭 시 즉시 ON/OFF.
🗺️
화면 04
코스 상세 — 내 코스
지도 반 + 정보 반. AppBar에 편집/삭제 드롭다운 메뉴.
⭐
화면 05
코스 상세 — 남의 코스
지도 반 + 정보 반. AppBar에 즐겨찾기 별 토글 + "내 코스로 복사" 메뉴.

화면 흐름

01 홈 메뉴 → 바이크 원 클릭 → 확장 메뉴 열림
라이딩 코스 클릭 → 02 내 코스 탭 ↔ 03 탐색 탭
아이템 클릭 → 04 상세 (내 코스) 편집/삭제 메뉴 포함
남의 코스 클릭 → 05 상세 (남의 코스) 즐겨찾기 토글 + 복사 메뉴

주의사항 / 결정 필요 사항

flutter-dev 참고사항

- SafeArea: 이 화면은 shell 밖 전체 화면이므로 시스템 하단바 오버랩 방지 필요. 스크롤 리스트 bottom padding에 MediaQuery.of(context).viewPadding.bottom 추가 필수.

- 지도 높이: 상세 화면 지도 영역을 고정 340px으로 표현했으나, 실기기 화면 크기에 따라 비율 기반(MediaQuery.of(context).size.height * 0.45)으로 조정 가능. flutter-dev 판단 위임.

- FAB + 스크롤: 내 코스 탭 FAB(+) 클릭 시 "코스 생성은 2차 스코프" 안내 토스트 표시 (목업 확인 완료 후 2차에서 화면 추가).

- 확장 메뉴 4버튼 간격: 기존 3버튼 sideSpread=70에서 4버튼으로 늘어남. 목업 기준 좌우 10px/274px, 내측 88px/196px. 좌우 대칭이므로 실기기에서 겹침 없는지 확인 필요.

- 탐색 탭 검색: 목업은 in-memory 필터 즉시 반응 패턴(PlaceSearchAddScreen과 동일). 서버 왕복 없이 클라이언트 측 필터 권장.

- 코스 생성/편집 (#15, #16): 2차 스코프. FAB 버튼은 배치하되, 탭 시 "준비 중" 다이얼로그 또는 disabled 처리.
